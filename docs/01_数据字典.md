# ChainPilot 数据字典

> 版本：v1.0 ｜ 所有表落在 `data/chainpilot.duckdb`
> 本文档同时是 NL→SQL Agent 的领域知识注入源：§4 业务术语表会被序列化后进入 prompt。

## 1. 数据来源与场景设定

**场景**：一家虚构的消费品制造商 "Contoso Foods"，生产 ~100 个成品 SKU（对应 M5 FOODS 类目真实销量），每个成品由 3~8 种物料组装/加工而成，物料由多家供应商按份额供货。计划员需要：预测成品需求 → 展开物料需求 → 监控库存 DOI → 识别缺料风险。

| 来源 | 表 | 说明 |
|---|---|---|
| Kaggle M5（真实） | sales_daily, calendar, prices | 降采样：FOODS_3 类目、CA 州门店销量按 SKU 聚合到全国单序列，取最后 3 年 |
| 模拟生成器（`data/scripts/generate_sim.py`） | products, bom, materials, suppliers, supply_split, inventory_onhand, open_po | 固定随机种子，规则可解释，含注入的 ground-truth 缺料场景 |
| 引擎写回 | forecast_daily, forecast_metrics, material_risk | M2/M3 产物 |

## 2. 表结构

### 2.1 M5 侧（真实数据）

**sales_daily** — 成品日销量（= 出货需求）
| 字段 | 类型 | 说明 |
|---|---|---|
| sku_id | VARCHAR PK(复合) | 成品编号，沿用 M5 item_id，如 `FOODS_3_090` |
| date | DATE PK(复合) | 日期 |
| units_sold | INTEGER | 当日销量（CA 州门店求和） |

**calendar** — 日历与事件
| 字段 | 类型 | 说明 |
|---|---|---|
| date | DATE PK | 日期 |
| weekday | VARCHAR | 星期 |
| event_name | VARCHAR NULL | 节假日/事件（M5 原字段） |
| is_weekend | BOOLEAN | 派生 |

**prices** — 售价（周粒度）
| 字段 | 类型 | 说明 |
|---|---|---|
| sku_id | VARCHAR | FK→products |
| week_start | DATE | 周起始日 |
| sell_price | DECIMAL(8,2) | 周售价 |

### 2.2 模拟制造层

**products** — 成品主数据
| 字段 | 类型 | 说明 |
|---|---|---|
| sku_id | VARCHAR PK | 与 M5 item_id 一致 |
| product_name | VARCHAR | 生成的可读名 |
| product_family | VARCHAR | 产品族（按 M5 dept 映射，如 `SNACKS`、`BEVERAGE`） |

**materials** — 物料主数据
| 字段 | 类型 | 说明 |
|---|---|---|
| material_pn | VARCHAR PK | 物料料号，格式 `PN-XXXXX` |
| material_name | VARCHAR | 可读名 |
| commodity | VARCHAR | 商品类别：`PACKAGING` / `RAW_FOOD` / `ADDITIVE` / `LABEL` / `CONTAINER` |
| item_group | VARCHAR | 物料组（commodity 下二级分组，风险聚合维度之一） |
| unit_cost | DECIMAL(8,4) | 单位成本 |

**bom** — 物料清单（单层，MRP-lite）
| 字段 | 类型 | 说明 |
|---|---|---|
| sku_id | VARCHAR PK(复合) | FK→products |
| material_pn | VARCHAR PK(复合) | FK→materials |
| qty_per_unit | DECIMAL(8,3) | 单台用量 |

生成规则：每 SKU 挂 3~8 条；**20% 物料为多 SKU 共用料**（制造"一料短缺、多品受累"的传导场景）；同 product_family 共用率更高。

**suppliers** — 供应商主数据
| 字段 | 类型 | 说明 |
|---|---|---|
| supplier_id | VARCHAR PK | `SUP-XXX` |
| supplier_name | VARCHAR | 虚构名 |
| region | VARCHAR | `CN` / `SEA` / `US` / `EU` |

**supply_split** — 供应份额（复刻 Share Split 概念）
| 字段 | 类型 | 说明 |
|---|---|---|
| material_pn | VARCHAR PK(复合) | FK→materials |
| supplier_id | VARCHAR PK(复合) | FK→suppliers |
| split_pct | DECIMAL(5,2) | 份额，同一 PN 合计=100 |
| lead_time_days | INTEGER | 该供应商供货提前期，7~90 天 |
| moq | INTEGER | 最小起订量 |

生成规则：60% 物料双源、30% 单源（风险因子）、10% 三源；单源物料是缺料场景的主要注入点。

**inventory_onhand** — 库存快照
| 字段 | 类型 | 说明 |
|---|---|---|
| material_pn | VARCHAR PK(复合) | FK→materials |
| snapshot_date | DATE PK(复合) | 快照日（周频，最新快照 = 当前库存） |
| qty_onhand | INTEGER | 现有量 |

**open_po** — 在途采购单
| 字段 | 类型 | 说明 |
|---|---|---|
| po_id | VARCHAR PK | `PO-XXXXXX` |
| material_pn | VARCHAR | FK→materials |
| supplier_id | VARCHAR | FK→suppliers |
| qty | INTEGER | 订单量 |
| eta_date | DATE | 预计到货日 |

### 2.3 引擎写回表

**forecast_daily** — 需求预测（M2 产物）
| 字段 | 类型 | 说明 |
|---|---|---|

| date | DATE PK(复合) | 未来 28 天 |
| model_name | VARCHAR PK(复合) | `seasonal_naive` / `ets` / `lightgbm` |
| yhat | DECIMAL(10,2) | 预测销量 |

**forecast_metrics** — 回测指标（M2 产物，简历数字来源）
| 字段 | 类型 | 说明 |
|---|---|---|
| model_name | VARCHAR | |
| fold | INTEGER | 滚动回测折 |
| mape / wrmsse | DECIMAL | 分模型分折指标 |

**material_risk** — 物料风险分级（M3 产物）
| 字段 | 类型 | 说明 |
|---|---|---|
| material_pn | VARCHAR PK(复合) | |
| eval_date | DATE PK(复合) | 评估日 |
| doi_days | DECIMAL(6,1) | 库存天数 |
| lt_coverage | DECIMAL(5,2) | DOI / 有效提前期 |
| supplier_concentration | DECIMAL(5,2) | 最大单一供应商份额 |

## 3. 派生视图与核心计算

```sql
-- v_material_demand_daily：MRP-lite 展开（物料日需求 = Σ 成品预测 × 单台用量）
SELECT b.material_pn, f.date, SUM(f.yhat * b.qty_per_unit) AS demand_qty
FROM forecast_daily f JOIN bom b USING (sku_id)
WHERE f.model_name = 'lightgbm' GROUP BY 1, 2;

-- DOI = 最新现有库存 / 未来28天日均物料需求
-- 预计库存余额（projected balance）= 现有库存 + 累计在途到货 - 累计需求，逐日滚动
-- gap_date = 余额首次 < 0 的日期；gap_qty = 期末最大缺口
```

**风险分级规则**（M3 实现，规则即文档）：
| 等级 | 条件（或逻辑命中即升级） |
|---|---|
| RED | 预测期内出现缺口（gap_qty>0）且首个断料日 < 最短 lead time（追料已来不及） |
| ORANGE | 有缺口但断料日 > lead time（可追料）；或 DOI < 0.5×lead time 且单源供应 |
| YELLOW | DOI < lead time；或供应商集中度 > 70% 且 DOI < 1.5×lead time |
| GREEN | 以上皆不命中 |

**ground truth 注入**：生成器刻意构造 8~12 个已知缺料场景（单源+低库存+在途不足等组合），写入 `data/ground_truth_scenarios.json`，M3 验收 = 风险引擎对这些场景的召回率 100%。

## 4. 业务术语表（Agent 领域知识，序列化进 prompt）

| 术语 | 定义 | 计算式 / 落点 |
|---|---|---|
| DOI (Days of Inventory) | 现有库存可支撑的天数 | `qty_onhand / avg(未来28天日均需求)` → material_risk.doi_days |
| 缺料暴露 (Shortage Exposure) | 预计库存余额转负 | material_risk.gap_qty / gap_date |
| LT 覆盖率 | DOI 与提前期之比，<1 意味着断料前补不上货 | doi_days / min(lead_time_days) |
| Share Split | 一个物料在多家供应商间的份额分配 | supply_split.split_pct |
| 单源风险 | 物料仅一家供应商供货 | count(supplier)=1 per material_pn |
| 供应商集中度 | 最大单一供应商份额 | max(split_pct) per material_pn |
| MRP 展开 | 成品需求 × BOM 用量 → 物料需求 | v_material_demand_daily |
| 共用料 | 被多个 SKU 引用的物料，短缺时多品受累 | count(sku)>1 per material_pn in bom |
| 在途 (Open PO) | 已下单未到货的采购量 | open_po（eta_date 未到） |
| Commodity / Item Group | 物料的一级/二级分类，风险聚合维度 | materials.commodity / item_group |

> 注：术语概念与制造业通用叫法一致（DOI、Share Split、MRP 均为行业标准词），不含任何联想内部数据或专有信息。

## 5. 数据规模预估

| 表 | 行数量级 |
|---|---|
| sales_daily | ~100 SKU × 1095 天 ≈ 11 万行 |
| bom | ~550 行；materials ~300；suppliers ~40；supply_split ~520 |
| inventory_onhand | ~300 PN × 156 周 ≈ 4.7 万行 |
| open_po | ~1500 行 |
| forecast_daily | 100 × 28 × 3 模型 ≈ 8400 行/次 |
| material_risk | ~300 行/评估日 |
