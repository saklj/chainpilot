## 2. 评测集设计（`evals/questions_50.jsonl`）

- **50 题 = 20 模板题 + 20 开放题 + 10 对抗题**；schema 覆盖全部 13 表 + 4 视图
- 4 题来自产品负责人真实试用（`source: "user_testing"`），包括全场最难的混合意图题（"DOI 是什么意思，我现在想看缺料"）
- 对抗题谱系：诱导编造 ×4、prompt 注入 ×3、越权/隐私 ×2、闲聊越界 ×1
- **判分是确定性代码**（5 种 check 类型），无 LLM 裁判：
  - `exact_set` / `superset`：实体集合比对，**gold 多列皆为等价答案键**（id/名称同指一物）
  - `contains_values`：关键单元格包含（数值 0.5% 容差；问题中已给出的实体不要求回显）
  - `refusal`：对抗题以 `refused=True` 为检出
  - `answer_keywords`：术语解释题的关键词/禁词/附加集合校验
- 纪律：温度 0、不重试、首次结果即判分；指标不达标只修产品或修有据可查的出题缺陷，不放宽判分口径

## 3. 迭代史（4 轮，80% → 92.5%）

| 轮 | SQL 正确率 | 主要动作 |

| 2 | 75.0% | Codex 修复豁免后失败题换了一批——暴露判分对合理变体太脆 + 模型温度 0 仍抖动。Codex 满两轮，Claude 接手 |

## 4. 护栏假阳性家族（面试深挖素材）

护栏原则"答案中每个数字必须能在 SQL 结果集回查"在实战中撞上的全部误伤类别与对策：

| 类别 | 例子 | 对策 |
|---|---|---|
| 结果字符串内嵌数字 | "Carton 271" | 结果集字符串单元格整体掩码（裸 "271" 仍拦） |
| 我们自己 prompt 里的数字 | 术语表公式"未来28天"、截断说明"前 50 行" | `exempt_text` 参数：自有 prompt 中的数字不算模型新主张 |
| 展示结构数字 | 行数、序号"1."、"前 10"、"41 个" | 行数/序数/量词计数（≤行数）/Top-N 豁免 |
| 无年份日期变体 | "6月17日"、"05-25"、"5/24" | 月日格式族按 (月,日) 与证据日期比对 |
| 标识符改写 | "SUP-040"→"供应商 40" | 结果集标识符形单元格的数字部分豁免 |
| 比较阈值 | "缺口大于 0" | 仅紧邻比较词的字面 0 豁免 |

每类豁免均带反向测试（编造数字在任何上下文仍被拦）。**真阳性战绩**：拦下模型自行求和的合计数（周报与 T09 各一次）、拦下写错的断料日期（数据无 6月25日）。

## 5. 温度 0 ≠ 可复现（重要工程发现）

- DeepSeek v4-flash 温度 0 下，同一 prompt 的输出仍逐次漂移（措辞、日期格式、列选择、偶发格式错误），推测与 MoE 路由/推理批处理有关
- 因此：**评测指标以区间呈现**（85%~92.5%），单点数字不诚实；**产品数字全部由代码直出**（周报表格、护栏比对），LLM 只负责措辞——这是 D6"护栏是代码不是 prompt"的又一层论据
- 判分器对合理变体（同义列、等价标识、日期格式）的鲁棒化把跑分抖动从 ±7.5pt 收敛到 ±3.75pt

## 6. 剩余失败题定性（封版轮，6/40）

- T05/O07：并列/排序类问题模型未穷尽或排序键不同——理解偏差，真实失败
- T17：把"事件日类型"按日分组而非按类型——真实失败
- O01/O12：护栏边界抖动（同题多轮有过有挂）——可用性代价，如实呈现
- O18：偶发输出格式错（sql_rejected/generation_failed）——不重试纪律下的流失
- 结论：不再针对性追修——继续修就是过拟合评测集；剩余失败进 M6 复盘

## 7. 选型问答（面试预案）

- **为什么护栏是代码不是 prompt？** prompt 层"不许编造"我们写了，模型照样自己求和（T09 实录）；只有确定性回查能给出可审计的拦截证据。AssessMate 结论在第二个项目复现。
- **为什么判分不用 LLM 当裁判？** 裁判也会幻觉且不可复现；确定性判分让每个失败题都能人工复盘定性（本报告 §3 的迭代全靠这个）。
- **85% 之外的 15% 怎么办？** 产品层有三重兜底：护栏拦截（不出错答案）、拒答语义（明确说不知道）、蓝图降级预案（高频问题模板化保 100%）。
